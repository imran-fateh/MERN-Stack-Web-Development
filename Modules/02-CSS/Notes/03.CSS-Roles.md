## 🎯 CSS Rules & Specificity Notes

CSS Rules control how styles are applied to HTML elements. Understanding CSS Rules and Specificity is essential for writing clean and professional stylesheets.

---

# 📖 What are CSS Rules?

CSS Rules are used to apply styles to HTML elements.

A CSS Rule consists of a selector and one or more declarations.

# ❓ Why Do We Use CSS Rules?

CSS Rules help control the appearance of a webpage, including:

- 🎨 Colors
- 🔤 Fonts
- 📏 Spacing
- 🖼 Borders
- 📐 Layouts

---

# 📝 CSS Rule Syntax

selector {
property: value;

}

# 💻 Example

h1 {
color: blue;

}

# 🔍 Explanation

The "h1" selector targets all heading elements and changes their text color to blue.

---

# 🔄 Last Rule Wins

When multiple CSS Rules target the same element with equal priority, the last rule is applied.

# 💻 Example

h1 {
color: red;

}

h1 {
color: blue;

}

h1 {
color: green;

}

# ✅ Result

Green Color Applied

# 🔍 Explanation

All selectors have the same specificity, so the browser applies the last rule.

# ⚠ Important Notes

- Order matters when specificity is equal.
- The last declared rule overrides previous rules.

# 📝 Summary

When selectors have equal priority, the last rule wins.

---

# 🎯 Specificity

Specificity determines which CSS Rule has higher priority.

# 💻 Example

<h1 class="heading" id="title">
    CSS Roles
</h1>

h1 {
color: blue;

}

.heading {
color: purple;

}

#title {
color: green;

}

# ✅ Result

Green Color Applied

# 🔍 Explanation

The ID Selector has higher priority than the Class Selector and Element Selector.

# ⚠ Important Notes

- ID selectors have greater specificity.
- Class selectors have medium specificity.
- Element selectors have lower specificity.

# 📝 Summary

Specificity decides which rule the browser applies.

---

# 🏷 CSS Priority Order

Priority from lowest to highest:

Element Selector
↓
Class Selector
↓
ID Selector
↓
!important

# 📝 Summary

Higher specificity rules override lower specificity rules.

---

# ⚡ !important Rule

The "!important" keyword gives a CSS property the highest priority.

# 💻 Example

h1 {
color: wheat !important;

}

# 🔍 Explanation

The browser applies this property even if other selectors have higher specificity.

# ⚠ Important Notes

- Use "!important" only when necessary.
- Excessive usage makes CSS difficult to maintain.
- Professional developers avoid overusing it.

# ❌ Common Mistakes

- Using "!important" everywhere.
- Ignoring proper specificity.

# 📝 Summary

The "!important" rule overrides almost all other CSS rules.
